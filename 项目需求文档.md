# 建筑师助手应用 - 项目需求文档

## 一、项目概述

**项目名称**：建筑师助手（Architect Assistant）

**项目描述**：一个面向建筑师的移动端辅助工具，帮助建筑师进行项目管理、设计灵感收集、材料管理和团队协作。

**技术栈**：
- 前端：UniApp（支持多端发布）
- 后端：FastAPI（Python）
- 数据库：SQLite/PostgreSQL

---

## 二、核心功能模块

### 1. 用户管理模块
- **用户注册/登录**
  - 支持手机号/邮箱注册
  - 密码登录
  - Token认证机制
  
- **用户信息管理**
  - 个人资料编辑（姓名、头像、公司、职位等）
  - 专业认证信息

### 2. 项目管理模块
- **项目列表**
  - 查看所有项目
  - 项目状态筛选（进行中、已完成、暂停）
  - 项目搜索功能
  
- **项目详情**
  - 项目基本信息（名称、地址、面积、预算等）
  - 项目进度跟踪
  - 项目文档管理
  - 项目图片/设计图管理
  
- **项目创建/编辑**
  - 创建新项目
  - 编辑项目信息
  - 删除项目

### 3. 设计灵感库
- **灵感收集**
  - 拍照上传
  - 图片导入
  - 添加标签和描述
  
- **灵感分类**
  - 按类型分类（外观、室内、景观、细节等）
  - 按风格分类（现代、古典、工业风等）
  - 自定义标签
  
- **灵感浏览**
  - 瀑布流展示
  - 详情查看
  - 收藏功能

### 4. 材料管理模块
- **材料库**
  - 材料列表展示
  - 材料详情（名称、规格、价格、供应商等）
  - 材料图片
  
- **材料搜索**
  - 按名称搜索
  - 按类型筛选
  - 按价格范围筛选
  
- **供应商管理**
  - 供应商信息
  - 联系方式
  - 历史合作记录

### 5. 团队协作模块
- **团队成员管理**
  - 添加团队成员
  - 成员角色分配
  - 成员权限管理
  
- **任务分配**
  - 创建任务
  - 分配给团队成员
  - 任务状态跟踪
  
- **消息通知**
  - 项目更新通知
  - 任务提醒
  - 团队消息

### 6. 文档管理模块
- **文档上传**
  - 支持多种格式（PDF、DWG、图片等）
  - 文档分类
  
- **文档浏览**
  - 在线预览
  - 下载功能
  
- **版本管理**
  - 文档版本记录
  - 版本对比

## 四、后端API接口设计

### 1. 用户相关接口
```
POST   /api/auth/register          # 用户注册
POST   /api/auth/login             # 用户登录
GET    /api/auth/profile           # 获取用户信息
PUT    /api/auth/profile           # 更新用户信息
```

### 2. 项目相关接口
```
GET    /api/projects               # 获取项目列表
POST   /api/projects               # 创建项目
GET    /api/projects/{id}          # 获取项目详情
PUT    /api/projects/{id}          # 更新项目
DELETE /api/projects/{id}          # 删除项目
POST   /api/projects/{id}/images   # 上传项目图片
```

### 3. 灵感相关接口
```
GET    /api/inspirations           # 获取灵感列表
POST   /api/inspirations           # 添加灵感
GET    /api/inspirations/{id}      # 获取灵感详情
PUT    /api/inspirations/{id}      # 更新灵感
DELETE /api/inspirations/{id}      # 删除灵感
GET    /api/inspirations/tags      # 获取标签列表
```

### 4. 材料相关接口
```
GET    /api/materials              # 获取材料列表
POST   /api/materials              # 添加材料
GET    /api/materials/{id}         # 获取材料详情
PUT    /api/materials/{id}         # 更新材料
DELETE /api/materials/{id}         # 删除材料
GET    /api/suppliers              # 获取供应商列表
```

### 5. 团队相关接口
```
GET    /api/teams                  # 获取团队列表
POST   /api/teams                  # 创建团队
GET    /api/teams/{id}/members     # 获取团队成员
POST   /api/teams/{id}/members     # 添加团队成员
GET    /api/tasks                  # 获取任务列表
POST   /api/tasks                  # 创建任务
PUT    /api/tasks/{id}             # 更新任务状态
```

### 6. 文件上传接口
```
POST   /api/upload/image           # 上传图片
POST   /api/upload/document        # 上传文档
GET    /api/files/{id}             # 获取文件
```

## 五、数据库设计

### 主要数据表

1. **users** - 用户表
   - id, username, email, password_hash, phone, avatar, company, position, created_at

2. **projects** - 项目表
   - id, user_id, name, address, area, budget, status, description, created_at, updated_at

3. **project_images** - 项目图片表
   - id, project_id, image_url, description, created_at

4. **inspirations** - 灵感表
   - id, user_id, title, image_url, description, tags, category, style, created_at

5. **materials** - 材料表
   - id, name, category, specification, price, unit, supplier_id, image_url, description

6. **suppliers** - 供应商表
   - id, name, contact, phone, email, address, description

7. **teams** - 团队表
   - id, name, creator_id, description, created_at

8. **team_members** - 团队成员表
   - id, team_id, user_id, role, joined_at

9. **tasks** - 任务表
   - id, project_id, title, description, assignee_id, status, due_date, created_at

## 六、非功能性需求

### 1. 性能要求
- 接口响应时间 < 500ms
- 图片加载优化（缩略图、懒加载）
- 支持离线缓存

### 2. 安全要求
- JWT Token认证
- 密码加密存储
- 文件上传安全校验
- API访问频率限制

### 3. 兼容性要求
- 支持iOS、Android平台
- 支持微信小程序
- 响应式设计

### 4. 用户体验
- 简洁美观的UI设计
- 流畅的交互动画
- 友好的错误提示
- 加载状态反馈
