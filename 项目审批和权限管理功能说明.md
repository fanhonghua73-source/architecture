# 项目审批和权限管理功能说明

## 功能概述

实现了完整的项目审批流程和权限管理系统，确保项目创建需要审批，同时已审批用户可以查看所有已批准的项目。

## 已实现功能

### 1. 项目审批状态
- ✅ `pending` - 待审批（普通用户创建的项目默认状态）
- ✅ `approved` - 已批准（root用户创建的项目自动批准）
- ✅ `rejected` - 已拒绝

### 2. 权限控制

#### 项目查看权限
- **Root用户**：可以查看所有项目
- **已审批用户**：可以查看所有已批准的项目（只读）
- **普通用户**：只能查看自己创建的项目

#### 项目编辑权限
- **项目创建者**：可以编辑自己的项目
- **Root用户**：可以编辑所有项目
- **已审批用户**：只能查看，不能编辑

#### 项目删除权限
- **项目创建者**：可以删除自己的项目
- **Root用户**：可以删除所有项目

### 3. 审批流程

#### 创建项目
1. 普通用户创建项目 → 状态为"待审批"
2. Root用户创建项目 → 状态为"已批准"（自动批准）

#### 审批项目
1. Root用户进入"我的" → "项目审批"
2. 查看待审批项目列表
3. 点击"批准"或"拒绝"按钮
4. 项目状态更新

### 4. 前端页面

#### 项目列表页面
- 显示审批状态标签（待审批/已批准/已拒绝）
- 显示项目状态标签（进行中/已完成/暂停）
- 已审批用户可以看到所有已批准的项目

#### 项目详情页面
- 显示审批状态
- 只有创建者和root可以编辑
- 已审批用户只能查看，不能编辑
- 状态选择器只对有编辑权限的用户可用

#### 项目审批页面（仅root）
- 显示所有待审批项目
- 可以批准或拒绝项目
- 可以查看项目详情

## 数据库变更

### Project表新增字段
```sql
approval_status VARCHAR DEFAULT 'pending'
  - pending: 待审批
  - approved: 已批准
  - rejected: 已拒绝
```

## API接口

### 项目管理
```
GET    /api/projects                    # 获取项目列表（根据权限过滤）
POST   /api/projects                    # 创建项目（自动设置审批状态）
GET    /api/projects/{id}               # 获取项目详情（权限检查）
PUT    /api/projects/{id}               # 更新项目（仅创建者和root）
DELETE /api/projects/{id}               # 删除项目（仅创建者和root）
```

### 项目审批
```
GET    /api/projects/pending/list       # 获取待审批项目（仅root）
POST   /api/projects/{id}/approve       # 审批项目（仅root）
       参数: approved=true/false
```

## 使用说明

### Root用户操作

#### 审批项目
1. 登录系统
2. 进入"我的"页面
3. 点击"项目审批"
4. 查看待审批项目列表
5. 点击"批准"或"拒绝"按钮
6. 确认操作

#### 查看所有项目
- 在"项目"页面可以看到所有项目
- 包括待审批、已批准、已拒绝的项目

### 普通用户操作

#### 创建项目
1. 进入"项目"页面
2. 点击右上角"+"按钮
3. 填写项目信息
4. 提交后状态为"待审批"
5. 等待root用户审批

#### 查看项目
- 只能看到自己创建的项目
- 可以编辑自己的项目
- 项目显示审批状态

### 已审批用户操作

#### 查看项目
- 可以看到所有已批准的项目
- 不能编辑其他人的项目
- 只能查看项目详情

#### 创建项目
- 创建的项目需要等待审批
- 可以编辑自己的项目
